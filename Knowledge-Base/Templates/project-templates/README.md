# Project Forms (FOR)

This folder contains all **standardized forms (FOR)** of the Meridian Group ecosystem. Each form corresponds to a document that must be completed in one or more phases of a project's lifecycle.

The forms are not bureaucracy — they are the project's memory. Without them, knowledge gets lost between people, chats, and uncontrolled versions of files.

> **Index maintenance (instruction for Claude):** Before instantiating a new FOR, verify here that the number is not taken. If a completely new type of FOR is created (without precedent in this index), assign the next available number and add its entry to the corresponding table before creating the file in the project.

## Usage Flow by Project Phase

```
INITIATION
  ├── FOR-001 Project Sheet           ← Before starting
  └── FOR-002 Project Charter         ← When formalizing with the sponsor

PLANNING
  ├── FOR-003 Requirements Matrix     ← When gathering needs
  ├── FOR-011 Communication Plan      ← When organizing stakeholders
  ├── FOR-014 RACI Matrix             ← When defining the team
  └── FOR-015 Task Plan               ← Generated by Claude at planning close (see instructions in template)

EXECUTION
  ├── FOR-007 Weekly Report           ← Every Monday (% progress comes from `completeness` in Logbook)
  ├── FOR-008 Risk Matrix             ← Update weekly
  ├── FOR-009 Change Request          ← When there are changes
  ├── FOR-010 Issues Log              ← When there are problems
  └── FOR-016 8D Resolution           ← Critical/recurrent incidents (IE-2-2-1-1)

QUALITY
  ├── FOR-006 QA Matrix               ← Per test cycle
  ├── FOR-004 Test Plan               ← Before final acceptance
  └── FOR-005 Certification Matrix    ← Before going to production

CLOSURE
  ├── FOR-012 Final Report            ← With sponsor signature
  └── FOR-013 Lessons Learned         ← In session with the team
```

## File Naming Convention

### In the Obsidian Vault (Projects/*/Documents/)

Forms instantiated in projects use full nomenclature to guarantee unique names and visibility in the graph:

```
FOR-[NNN]_[Project-Code]_[Project-Name]_[type].md
```

Examples:
- `FOR-015_IE-1-1-3_Odoo-CRM-Fintrust_task-plan.md`
- `FOR-002_PR-7-16_Payment-Confirmation-Agent_project-charter.md`
- `FOR-008_IE-3-2-3-1_File-Migration_risk-matrix.md`

> The templates in **this folder** retain the generic name (e.g., `FOR-015-task-plan.md`). When instantiating a FOR in a project, rename with the full format.

### In SharePoint (external documents)

```
FOR-[NNN] [Form Name] - [Project] - v[X.X].[ext]
```

Examples:
- `FOR-002 Project Charter - Sheriff Portal - v1.0.docx`
- `FOR-007 Weekly Report - Odoo Migration - S23 2025.md`

**Rule:** Never use "final", "definitive", or dates as a substitute for version.
