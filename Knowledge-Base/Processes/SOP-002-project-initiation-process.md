# SOP-002 — Project Initiation Process

**Standard process for starting projects within the Meridian Group strategic portfolio**

---

## 1. Purpose and Scope

### 1.1 Purpose

Establish the standard process for starting projects within the Meridian Group strategic portfolio, ensuring that all projects have a solid foundation, clear objectives, and formal authorization before proceeding to planning.

### 1.2 Scope

This SOP applies to all projects in the strategic portfolio, from initial identification to formal authorization to proceed to the planning phase.

### 1.3 References

- FOR-001: Project Sheet
- FOR-002: Project Charter
- FOR-014: RACI Matrix
- SOP-001: General Project Management Methodology
- SOP-003: Requirements Gathering and Management

---

## 2. Initiation Criteria

### 2.1 Project Origin

Projects can originate from:

- **Strategic initiatives**: Defined in the institutional scorecard
- **Operational needs**: Identified by area sponsors
- **Technology opportunities**: Proposed by the technical team
- **Regulatory requirements**: External mandates or compliance

### 2.2 Viability Criteria

Every project must meet at least 3 of the following criteria:

- **Strategic alignment**: Directly contributes to a scorecard objective
- **Technical feasibility**: Achievable with available resources and technology
- **Return on investment**: Positive ROI or significant qualitative benefit
- **Time urgency**: Required within the current fiscal period
- **Critical dependencies**: Enabler for other priority projects

## 3. Initiation Process

### 3.1 Step 1 — Identification and Request

**Responsible**: Project Sponsor.

**Activities**:
1. Identify opportunity or need
2. Complete initial request in Microsoft Planner
3. Assign temporary label: "Pre-Initiation"
4. Notify the Project Manager

**Deliverables**: Project request in Planner with basic description.

### 3.2 Step 2 — Initial Evaluation

**Responsible**: Project Manager + Sponsor.

**Activities**:
1. Review viability criteria
2. Consult with key stakeholders
3. Evaluate resource capacity
4. Determine relative priority

**Decision Criteria**:
- **Approved**: Meets criteria, proceeds to formal documentation
- **Rejected**: Does not meet criteria, archived with justification
- **Deferred**: Meets criteria, but insufficient capacity

### 3.3 Step 3 — Formal Documentation

**Responsible**: Project Manager with Sponsor support.

**Activities**:
1. Complete Project Sheet (FOR-001)
2. Develop initial Project Charter (FOR-002)
3. Identify key stakeholders
4. Create preliminary RACI matrix (FOR-014)
5. Estimate resources and initial duration

**Deliverables**:
- Project Sheet v1.0
- Project Charter v1.0
- RACI Matrix v1.0
- Resource estimation

### 3.4 Step 4 — Stakeholder Validation

**Responsible**: Project Sponsor.

**Activities**:
1. Present documentation to key stakeholders
2. Collect feedback and adjustments
3. Validate effort estimates with assigned resources
4. Confirm resource availability

**Validation Criteria**:
- Stakeholders confirm need and priority
- Resources confirm estimated availability
- Sponsor validates scope and objectives

### 3.5 Step 5 — Formal Authorization

**Responsible**: Strategic Committee.

**Activities**:
1. Present project at weekly Strategic Committee meeting
2. Review alignment with current portfolio
3. Evaluate impact on other projects
4. Make authorization decision

**Authorization Criteria**:
- Confirmed alignment with strategic objectives
- Resources available or clear release plan
- No critical conflicts with other projects
- Sponsor committed to project leadership

---

## 4. Roles and Responsibilities

### 4.1 Project Sponsor

- **Initiation**: Identify need and complete request
- **Documentation**: Provide detailed information for documents
- **Validation**: Present to stakeholders and obtain buy-in
- **Commitment**: Guarantee availability of resources from their area

### 4.2 Project Manager

- **Facilitation**: Guide the complete initiation process
- **Documentation**: Create and maintain formal documents
- **Evaluation**: Analyze technical and resource feasibility
- **Coordination**: Organize meetings and follow-ups

### 4.3 Strategic Committee

- **Prioritization**: Decide project execution order
- **Authorization**: Formally approve planning start
- **Resources**: Resolve resource conflicts between projects
- **Alignment**: Ensure coherence with organizational strategy

### 4.4 Key Stakeholders

- **Input**: Provide high-level requirements
- **Validation**: Confirm need and priority
- **Commitment**: Confirm availability to participate

---

## 5. Phase Exit Criteria

### 5.1 Completed and Approved Documents

- Project Sheet v1.0 signed by Sponsor
- Project Charter v1.0 approved by Strategic Committee
- RACI Matrix v1.0 validated with stakeholders
- Resource estimation confirmed with assignees

### 5.2 Obtained Authorizations

- Sponsor confirms project leadership
- Key resources confirm estimated availability
- Strategic Committee authorizes proceeding to planning
- Project Manager accepts facilitation responsibility

### 5.3 Tool Configuration

- Project created in Microsoft Planner with standard structure
- Project folders created in SharePoint
- Teams channel configured (if required)
- Initial meeting calendar established

## 6. Verification Points

### 6.1 Verifications for Project Manager

- [ ] Request received
- [ ] Viability criteria evaluated and documented
- [ ] Project Sheet completed and reviewed
- [ ] Project Charter developed with sponsor
- [ ] Stakeholders identified and contacted
- [ ] RACI Matrix created and validated
- [ ] Resource estimates obtained
- [ ] Presentation to Strategic Committee scheduled
- [ ] Authorization decision documented
- [ ] Tool configuration completed

### 6.2 Verifications for Sponsor

- [ ] Project need clearly articulated
- [ ] Specific and measurable objectives defined
- [ ] Key stakeholders identified and consulted
- [ ] Area resources committed
- [ ] Preliminary schedule acceptable
- [ ] Expected deliverables clear
- [ ] Success criteria defined
- [ ] Main risks identified

---

## 7. Escalation and Exceptions

### 7.1 Decision Escalation

- **Priority conflict**: Immediate escalation to Strategic Committee
- **Resources not available**: Escalation to the resource's sponsor
- **Significant scope changes**: Re-authorization by Strategic Committee

### 7.2 Emergency Projects

For critical projects requiring immediate start:

1. Verbal authorization from Committee with documentation within 48h
2. Initiation with minimum documents (Sheet + basic Charter)
3. Complete documentation during first week of planning
