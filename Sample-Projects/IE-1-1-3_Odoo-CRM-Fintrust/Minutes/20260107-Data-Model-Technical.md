# IE-1-1-3 · 2026-01-07 · Technical: Data Model Entity Definition

## Purpose

Review and close the Odoo data model entity definitions before the model validation session with Hugo Pacheco.

## Entity Model — Final

### 1. `fintrust.debtor` (Debtors)

Extends `res.partner` with:
- `debtor_code` (char, unique): Internal code assigned by Fintrust
- `debtor_type` (selection): Natural person / Legal entity / In-process
- `location_status` (selection): Located / Unlocatable / In process
- `employer` (char): Current employer (for garnishment)
- `monthly_income` (float): Declared monthly income
- `bank_account_ids` (one2many to `res.partner.bank`): List of associated bank accounts
- `contact_history_ids` (one2many): Change log of phones, emails, addresses

### 2. `fintrust.creditor` (Creditors)

Extends `res.partner` with:
- `creditor_code` (char, unique)
- `creditor_type` (selection): Bank / Finance company / Company / Individual
- `contract_ids` (one2many to `fintrust.creditor.contract`)

### 3. `fintrust.creditor.contract` (Creditor Contracts)

Independent model (not extension of an existing one):
- `creditor_id` (many2one to `fintrust.creditor`)
- `contract_code` (char, unique)
- `start_date` (date)
- `end_date` (date, optional for indefinite)
- `fee_percentage` (float): Percentage charged per recovered case
- `term` (integer): Days term for payment to creditor
- `commission_structure` (text): Description of how commissions are calculated
- `active` (boolean): To deactivate without deleting

### 4. `fintrust.lawyer` (Lawyers)

Extends `res.partner` with:
- `license_number` (char): Bar association number
- `specialty_ids` (many2many): Judicial areas (civil, commercial, labor)
- `court_assignments` (one2many): Courts where they operate (El Seibo, Santo Domingo, etc.)

### 5. `fintrust.case` (Legal Cases)

*Main model of non-judicial module.*
- `case_code` (char, unique, generated by sequence)
- `creditor_id` (many2one to `fintrust.creditor`)
- `case_type` (selection): Judicial / Extrajudicial / Pre-contentious
- `state` (selection): Use the extrajudicial flow states
- `receipt_date` (date): When the portfolio was received
- `assigned_lawyer_id` (many2one to `fintrust.lawyer`)
- `debtor_role_ids` (one2many to `fintrust.case.role`): Debtors linked to the case with their role
- `debt_product_ids` (one2many to `fintrust.case.debt.product`)
- `payment_agreement_ids` (one2many to `fintrust.agreement`)
- `bonita_state` (char, readonly): Judicial state brought from Bonita
- `total_debt` (float): Original debt amount
- `current_balance` (float): Updated balance

### 6. `fintrust.case.role` (Ganchi / Roles)

*Link table between Debtor and Case.*
- `case_id` (many2one to `fintrust.case`)
- `debtor_id` (many2one to `fintrust.debtor`)
- `role_type` (selection): Main debtor / Co-debtor / Guarantor / Spouse
- `active` (boolean)
- `assignment_date` (date)

### 7. `fintrust.case.debt.product` (Case Debts)

*Specific debts associated to a case.*
- `case_id` (many2one to `fintrust.case`)
- `product_name` (char): e.g., "Personal Loan 12345"
- `original_amount` (float): Amount at default date
- `updated_amount` (float): Current amount with interest and surcharges
- `debt_date` (date)
- `status` (selection): Active / Paid / Prescribed / In dispute

### 8. `fintrust.agreement` (Payment Agreements)

- `case_id` (many2one to `fintrust.case`)
- `agreement_type` (selection): Lump sum payment / Installments
- `total_agreed` (float)
- `installment_count` (integer)
- `installment_amount` (float)
- `start_date` (date)
- `status` (selection): Active / Completed / Defaulted / Cancelled
- `monitoring_history_ids` (one2many): Payment-by-payment record

### 9. `fintrust.activity` (Activities / Actions)

- `case_id` (many2one to `fintrust.case`)
- `activity_type` (selection): Call / Visit / Notification / Document delivery
- `description` (text)
- `result` (selection): Positive / Negative / Pending follow-up
- `responsible_id` (many2one to `res.users`)
- `activity_date` (datetime)
- `next_action_date` (date): For pending follow-up

### 10. `fintrust.note` (Notes / Comments)

- `case_id` (many2one to `fintrust.case`)
- `author_id` (many2one to `res.users`)
- `note_type` (selection): Internal / Client communication / Legal
- `content` (text)
- `date` (datetime)

## Technical Notes

1.  **Sequences:** All codes (`debtor_code`, `creditor_code`, `case_code`, `contract_code`) are auto-generated with `ir.sequence` in Odoo. No manual entry.

2.  **Historical contacts:** `contact_history_ids` is a JSON or one2many model that records changes. Decision: one2many model `fintrust.contact.history` with fields: `contact_type` (phone/email/address), `old_value`, `new_value`, `change_date`, `changed_by`.

3.  **Multi-company:** Not needed for MVP. All records are shared.

4.  **Security:** Two Odoo groups: `fintrust.group_admin` (full CRUD) and `fintrust.group_operator` (CRUD on cases and debtors, read-only on configuration).
