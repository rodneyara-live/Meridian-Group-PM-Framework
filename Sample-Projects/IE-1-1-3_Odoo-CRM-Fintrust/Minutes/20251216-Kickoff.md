# IE-1-1-3 · 2025-12-16 · Kickoff: Gap Analysis — Ledger-9 vs. Odoo CRM

## Context

The Fintrust project is NOT starting from scratch: they have a current CRM called **Ledger-9** that already manages the non-judicial part (debtors, creditors, lawyers, ganchi). The goal of this gap analysis is to map which Ledger-9 functionalities transfer to Odoo v17 and which must be built from scratch.

## Ledger-9 System Map

**Coverage of Ledger-9:**

- Debtors (CRUD fields + case history)
- Creditors (CRUD + contracts per creditor)
- Ganchi (CRUD + case history)
- Lawyers (CRUD + specialization)
- Cases: at the case level; case types: Judicial, Administrative, and Pre-Contentious. Within a case there are Legal Actions (subcases). Each action has its own state.

## Identified Gaps and Decisions

The following stand out from the analysis:

1.  **CASE-CENTRIC model:** In Ledger-9 it is already case-centric. In Odoo it MUST also be case-centric. Non-negotiable.

2.  **Ledger-9 is NO LONGER in use:** The team already stopped using it. All current records entered by the team will go directly into Odoo. Mass migration is NOT planned at this point. Just the catalog export to avoid re-typing.

3.  **Bonus/Incentive management:** Ledger-9 had a table for this. For MVP it stays in parking lot. It will be evaluated after closure of the first 3 modules.

4.  **Legal Actions vs. Cases:** In Ledger-9 they are different entities. In Odoo they will maintain this separation: Case (parent) → Legal Action (child, with its own state). Bonita will manage Legal Actions—Odoo will display them in read-only mode.

5.  **Hugo Pacheco role:** The only judicial specialist in the company. Must validate the data model and UAT tests before going to production.

6.  **Massive assignment of cases:** Ledger-9 had a feature for this. In Odoo it is done via Kanban. For MVP it is discarded.

7.  **Bank accounts per debtor:** List of bank accounts per debtor. This IS included in the MVP.
