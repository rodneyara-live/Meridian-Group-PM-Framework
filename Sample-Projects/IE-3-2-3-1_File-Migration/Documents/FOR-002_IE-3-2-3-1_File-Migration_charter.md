## Purpose and Justification

| Field | Information |
| --- | --- |
| **Project Purpose** | Outsource Meridian Group's file department management by contracting a specialized external provider, optimizing the handling, storage and custody of the company's physical documentation. |
| **Business Justification** | The current file department generates significant operational costs in personnel, physical spaces and maintenance. The selected provider already provides services to Meridian Group and has the experience and capacity to manage this process more efficiently. Outsourcing will free up human resources and spaces that can be redirected to higher-value activities while maintaining professional document custody standards. |
| **Expected Benefits** | Reduction of operational costs in payroll and file department expenses; optimization and freeing up of physical spaces currently used for storage; leveraging the provider's specialized infrastructure and expertise; improved document control and traceability through professional document management systems; reduced operational risks through certified custody standards; service scalability according to future business needs. |

---

## Objectives

| Field | Information |
| --- | --- |
| **Main Objective** | Successfully migrate Meridian Group's internal file department operations to a specialized external provider's service, ensuring operational continuity, document traceability and cost optimization. |
| **Specific Objectives** | (1) Map and document the current file department process, identifying flows, volumes and document criticality. (2) Obtain and evaluate the provider's updated commercial proposal. (3) Design a gradual migration plan that minimizes operational risks and guarantees document availability during transition. (4) Establish document access, consultation and return protocols with the provider. (5) Define the affected personnel transition scheme, complying with applicable labor law. (6) Implement controls and performance metrics for the outsourced service. (7) Validate compliance with information security and confidentiality requirements. |

## Assumptions and Constraints

| Field | Information |
| --- | --- |
| **Assumptions** | (1) The provider has sufficient installed capacity to absorb Meridian Group's file volume. (2) The company will maintain competitive and stable rates during the contract term. (3) The provider has certifications and meets required security regulations. (4) Current department personnel will cooperate during the transition process. (5) Critical documents are properly identified and can be classified. (6) There will be no significant regulatory changes in document custody requirements during implementation. (7) Freed-up spaces can be reused for other business needs. |
| **Constraints** | (1) PERSONNEL IMPACT: Requires sensitive handling of file department structure changes, with strict compliance with applicable labor law. (2) OPERATIONAL CONTINUITY: Access to critical documents cannot be compromised at any point during the transition. (3) LEGAL COMPLIANCE: Must comply with Law 172-13 on personal data protection and Superintendence of Banks regulations. (4) TIMING: Migration must be coordinated with operational and regulatory cycles (accounting closings, audits). (5) Any scope change must be processed through FOR-009 Change Request. |

## Post-Migration Operational Model

> Decisions made at the formal project kickoff (2026-03-03). These definitions are binding for the design of the detailed migration plan.

### Scanning Policy — Critical Requirement

| Rule | Detail |
|---|---|
| **Quality standard** | All documents must be scanned at HL level before any physical transfer. Scanning is not optional nor post-migration. |
| **Quality control** | Scanning quality verification is a precondition for packing and transferring each batch |
| **Photocopies** | Documents that are already photocopies are not archived if a digital version is available |

### Document Categorization

| Category | Description | Destination |
|---|---|---|
| **Permanent local** | Original promissory notes (only when clients request them — exceptional volume); active documents in mortgage registration and chattel registry | Fireproof filing cabinet in office |
| **Transitory** | Everything else: notified proceedings, mass garnishments (post-scanning), digitized documents not in active process | Transitory filing cabinets → offsite archive weekly |

### Transitory Local File Structure

| Element | Description |
|---|---|
| **Cabinet 1 (fireproof)** | Permanent documents: active mortgage, original promissory notes |
| **Cabinets 2 and 3 (regular)** | Transitory file: documents awaiting weekly offsite archive dispatch |
| **Dispatch frequency** | Once per week (Monday or Friday) — the transitory file is completely emptied |
| **Entry flow** | Document arrives → registration with case number → placed in corresponding transitory cabinet |

### Destruction Policy (Pending Formal Decision)

| Element | Status |
|---|---|
| Destruction of notified/purified acts post-scanning (e.g., mass garnishments) | **Proposal — pending formal decision at a later stage** |
