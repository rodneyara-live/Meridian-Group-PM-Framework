**[2026-03-03]** Kickoff completed: post-migration operational model and document policies defined

**Status:** 🟣 In progress · **Completion:** 10%

**Executive summary:** The formal project kickoff was held with [[David-Carver]], [[Tomas-Navarro]], [[Elena-Torres]] and [[Rodney-Ramirez]]. Key operational policies for the new model were defined: document categorization (local vs. offsite archive), critical HL scanning requirement, transitory local file structure, and a March 30 deadline for the new operational logistics model.

**What happened:**
Kickoff session held on 2026-03-03 in the conference room. [[David-Carver]] led the definition of the operational model. The following decisions were made:

*Transport logistics:*
- Mandatory pre-work: separate originals from copies before any movement
- First phase: do not destroy originals (conservative opening policy)
- All transfers must be **organized and traceable**
- Confirm SLAs with offsite archive based on actual processing capacity

*Scanning policy — critical requirement:*
- Document scanning must be at HL level ("the key to everything")
- No document may migrate without prior HL-quality scanning
- Quality control on scanning is a precondition for any transfer

*Document categorization (what stays local vs. what goes to offsite archive):*
- **Remains local (exceptions):** active cases in mortgage registration and chattel registry; original promissory notes (specific exception when clients request them — very low volume)
- **Goes to offsite archive:** everything else — notified proceedings, mass garnishments, already digitized documents not in active process
- Predominant flow is outbound: "from here to there a lot, from there to here very little"

*Non-archiving of photocopies policy:*
- No document that is already a photocopy will be archived if a digital version exists

*Proposed future policy (pending formal decision):*
- Possible destruction of notified/purified acts after scanning (e.g., mass garnishments)
- Requires formal decision at a later stage — not implemented yet

*Transitory local file structure (approved):*
- 1 fireproof filing cabinet → permanent documents (mortgage, promissory notes)
- 2 regular filing cabinets → transitory file
- Weekly emptying: dispatch to offsite archive once per week (Monday or Friday)
- Flow: document arrives → registered with case number → goes to transitory filing cabinet → goes to offsite archive in next weekly dispatch

*Inventory and validation:*
- Validate via document inventory that nothing remains outside the inventory
- The inventory is the central traceability control

*Committed deadline:*
- **March 30, 2026**: new operational logistics model

*Next step agreed by [[Rodney-Ramirez]]:*
- Working session with the file team to operationalize decisions into a plan A, B, C with tracking
