## 🗂️ Proposed ABC — [[Elena-Torres]]

Logical structure of internal phases **before** executing with offsite archive:

**Phase 0 — Internal precondition:**
1. Validate detailed and classified inventory of all existing documents
2. Classify inventory: critical / operational / dead file / to migrate or not
3. Verify that 100% of documents to be moved are previously digitized
4. Define chain of custody, delivery procedure and transport logistics

**Phase 1 — Migration by priority (lowest to highest risk):**
- 1st batch: dead file
- 2nd batch: closed cases
- Last batch: active cases (the most sensitive, including guarantees)

**Phase 2 — Transitory file policy:**
- Define minimum number of cases per dispatch (avoid "sheet by sheet" shipments)
- Define frequency of offsite archive shipments
- Align with the local file policy already agreed at kickoff (weekly emptying)

**Reconfirmed premises from 2026-03-03 kickoff:**
- Only **originals** are migrated (not photocopies) — significantly simplifies volume
- Everything moved must be **previously digitized**

## 🚨 Risks / Observations

- 🟡 **File team perception:** personnel may interpret the project as elimination of their department. Communication must be done carefully before any execution.
- 🟡 **Liability for originals at offsite archive:** offsite archive disclaims responsibility for damage/loss of original documents. Evaluate if additional insurance policy is required.
- 🟡 **Synchronicity with other projects:** migration borders on asset survey (move) — align schedules.
